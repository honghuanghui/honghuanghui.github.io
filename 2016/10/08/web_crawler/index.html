网络爬虫应用教程

导读：

本文将讲述基于Python-scrapy爬虫的一种实际运用，主要是用来监视网站的动态情况，然后第一时间通知指定的用户并且将最新内容推送给用户。因为是应用教程，故在写此博文时，已默认读者具有一定的Python、网络通讯、scrapy、html、正则表达式、xpath、数据库、Linux等等的爬虫基础。如果是新手，也可在此阅读后，体会到爬虫乐趣。

一、需求分析：

1.1 目标：

监视学校网站的特定内容是否有更新，若有更新，则立刻做出反馈。

1.2 解决方案：

首先模拟登录进入学校官网——爬取目标信息——对比之前一次所爬取的信息——若有不同则将其存入数据库、并且通过邮箱发送给用户；若相同，则进入定时执行环节——定时无限循环

1.3 实现方法：

1.3.1 模拟登录：

可以利用抓包工具，或者是火狐、谷歌浏览器自带的开发者插件捕获包头的格式，以及cookies的内容等登录所需的数据。然后在scrapy中模拟出来，实现爬虫的模拟登录。

1.3.2 爬取目标信息：

首先查看目标网页的源文件，利用正则表达式和XPATH匹配出目标内容。值得注意的是，如果你要爬取的内容分布在几十，或几百，或上千页。那么此时你还需要匹配出下一步所要爬取的网址内容，然后构造出新网址进行爬取。所以不管多少页，你都不需要费心，只是时间和性能的问题了。

1.3.3 对比信息：

这个容易实现，方法也很多，我是把爬取的内容放入文本，第二次爬取后，再取出文本内容进行比较。如果有不同，则将新的内容覆盖旧的内容。

1.3.4 发送邮件和存入数据库：

利用stmp协议进去邮件的发送。所以需要申请一个网易或者是QQ等的邮箱，开启stmp功能，结合Python的库，就可以实现邮件的发送了。至于存入数据库，则是一个可选项，如果你想收集之前的所有数据，那么此项则是必要的。我用的是mongodb（一种非关系型数据库，个人觉得在这个场合使用很合适。），当然你也可以使用其他的数据库，例如MySQL。

1.3.5 定时执行：

我用的是Linux的crontab任务计划程序，它是一个强大的任务计划程序。通过它，很容易就能让你的爬虫乖乖听话！至于如何操作，后面会详细介绍。

1.4 总结

有了以上的方案，只需要跟紧步伐，一步步完成，那么你的爬虫会像一个电子宠物一样听你指挥。是不是很有趣呢！

2.6 执行定时任务

2.6.1 关于crontab的知识

使用权限 : 所有使用者
使用方式 :
crontab [ -u user ] file
crontab [ -u user ] { -l | -r | -e }
说明 :
crontab 是用来让使用者在固定时间或固定间隔执行程序之用，换句话说，也就是类似使用者的时程表。-u user 是指设定指定 user 的时程表，这个前提是你必须要有其权限(比如说是 root)才能够指定他人的时程表。如果不使用 -u user 的话，就是表示设定自己的时程表。

参数 :
crontab -e : 执行文字编辑器来设定时程表，内定的文字编辑器是 VI，如果你想用别的文字编辑器，则请先设定 VISUAL 环境变数来指定使用那个文字编辑器(比如说 setenv VISUAL joe)
crontab -r : 删除目前的时程表
crontab -l : 列出目前的时程表
crontab file [-u user]-用指定的文件替代目前的crontab。
时程表的格式如下 :
f1 f2 f3 f4 f5 program
其中 f1 是表示分钟，f2 表示小时，f3 表示一个月份中的第几日，f4 表示月份，f5 表示一个星期中的第几天。program 表示要执行的程序。
当 f1 为 * 时表示每分钟都要执行 program，f2 为 * 时表示每小时都要执行程序，其馀类推
当 f1 为 a-b 时表示从第 a 分钟到第 b 分钟这段时间内要执行，f2 为 a-b 时表示从第 a 到第 b 小时都要执行，其馀类推
当 f1 为 /n 时表示每 n 分钟个时间间隔执行一次，f2 为 /n 表示每 n 小时个时间间隔执行一次，其馀类推
当 f1 为 a, b, c,… 时表示第 a, b, c,… 分钟要执行，f2 为 a, b, c,… 时表示第 a, b, c…个小时要执行，其馀类推
使用者也可以将所有的设定先存放在档案 file 中，用 crontab file 的方式来设定时程表。
